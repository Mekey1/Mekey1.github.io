第一章 编程篇

1.1 概述

编写程序是程序员以及任何想从事计算机行业必不可免的事，它是计算机的语言，是构造现有计算机世界的一砖一瓦。而编程语言分了很多种，从层次结构可以划分为高级语言、汇编语言、机器语言等。

可是明明只有一个计算机为何有这么多语言呢？这就不得不谈历史，就像历史文字演变那样从甲骨文到金文、小篆等等，计算机也有属于它自己的历史。在最初时期我们发现可以通过操作一个庞然大物ENIAC，它是世界上第一台计算机，通过向其输入相应的数据，其内部可以产生计算后的输出结果。随后通过历史的演变，硬件以及软件上的优化，可以发现二进制用来保存数据对硬件支持是最高效的，因为硬件可以很容易的表现出两种状态即代表了二进制的0和1，也就形成了上面所说的机器语言。

但机器语言编写十分的不便，于是“偷懒”的人们为了能够编写更快的代码就在其上又新增了一种汇编语言，按照汇编语言的格式最后编写好的代码通过一系列的工具转换后就又成了机器语言，这样确实加速了计算机的发展，而且这种“偷懒”思维也一直延续着。后续形成了像C、python、Java等各种计算机语言，这些语言就更厉害了，只需要像现在写字一样把逻辑写好，通过其编译器、链接器等工具就自动完成了一个程序的全部过程。而我们接下来将主要以C为主，因为其相对于其他语言而言“偷懒”的少，所以也可以学习到很多计算机底层逻辑。

1.2.4 变量类型

上面我们写的第一个程序main函数打印hello word，其中main的前面有个int，在最后有个return 0，这都是什么东西呢？这些就是编程语言中的类型，因为其前面要求是int型也就是整型，所以后续需要通过return返回一个整型值，我这里填的是0。类型有很多种，像整型、数组类型、字符型、浮点型等。

到这里我们就都明白了，我们自己写好一个函数或者给别人用的接口，通过定义好返回的类型值这样在函数运行结束时返回给调用者就可以了。

可是，为什么有那么多类型呢？因为在计算机中同一个数据有歧义，就像我们说的汉语老子，如果我们不加任何约定就说老子，到底是指父亲呢还是指道教文化的老子呢？这个就不得而知了，所以在写好函数的时候要规定好到底返回什么类型，这样调用函数的时候才不会理解错返回值的含义。当然大部分情况是需要返回值的，如果你编写的函数不需要返回任何东西，可以使用void代替，这样在函数运行结束时不用通过return返回任何值。后续我们在编写代码时根据具体情况使用不同类型的值进行操作。

接下来请将下面的代码编写出来

#include"stdio.h"

int main()
{
    int a = 0;
    char b = 'a';
    char c[] = "123";
    float d = 1.414;

    printf_s("a=%d b=%c c=%s d=%f", a, b, c, d);

    return 0;
}

注意：变量就是我们数学中所学到的x、y，使用它代指后面赋的值，这样使用到的变量就可以直接认为是后面的值，方便记忆和操作

我们可以发现上面的代码仅仅是在之前的基础上增加了各种类型的变量， 并将其打印出来。这里小伙伴要注意，赋值在编程语言中是使用‘=’符号，将右边的数据赋值给左边的变量。所以上面我们可以看出，我们一共定义了四个变量，a、b、c、d分别是整型int、字符类型char、字符数组类型char[]、浮点类型float。具体大家可以看下表：

类型含义
int有符号整数型，平时用到的数学中的整数就可以用来存放这个
char字符类型，一次只能存放一个字符
char[]字符数组类型，什么类型的数组就可以用来存放多个同类型的数据

最后通过调用别人写好的打印函数printf_s将这些数据打印出，同样的你用人家printf_s函数，就要遵守它的规范，它明确要求了在输出内容时%d就是按照十进制输出，%c是按照字符输出，%s是按照字符串输出，%f按照浮点输出，%x按照十六进制输出等。想要学习更多官方函数的使用规范，记得参考微软的官方文档：printf_s、_printf_s_l、wprintf_s、_wprintf_s_l | Microsoft Learn

这里肯定又要问版本位数是什么？Windows主要把版本分为32位和64位，这里我们暂且认为版本的位数对于程序能够在内存中存放数据的最大值是有关系的。之前我们已经提到利用二进制对硬件很友好，而内存上的地址需要通过数进行标记，所以32位就是2的32次方个数，这样这些地址就全部被标记好了，但是如果程序想要更大的空间32位显然不够，所以才有了后来的64位。

我们已经清楚了计算机的设计就是为了让我们能够“偷懒”，可现在这种重复性的工作又得让我们去做，实在是太傻了。所以必须有个东西能够解决重复性的问题，这个时候我们就需要再学习一种新的语法，就是循环结构。请编写好如下的代码：

#include"stdio.h"

int main()
{
    int i = 0;
    for (i; i < 10; i++)
    {
        printf_s("111\n");
    }

    while (i--)
    {
        printf_s("222\n");
    }

    do
    {
        printf_s("333\n");
    } while (i);

    return 0;
}

上面代码我一共写了最常用的三个循环结构，分别是for循环，while循环，do-while循环结构，还有一个奇怪的东西就是i++和i--，接下来我就一一解释这些都是什么。在for循环中，第二个分号处是判断句，如果结果非0就可以进入循环体内，也就是大括号内执行代码。while循环后面的括号只要为非0就可以进入循环体内，do-while循环先执行一次循环体，然后再判断while循环。为了方便记忆列出下表：

循环结构含义
for循环第一个分号前进行赋值，第二个分号前进行判断是否非0，非0则进入循环体，第三个分号前是循环体运行结束后要执行的内容
while循环括号内只要为非0就可以进入循环体内
do-while循环先执行一次循环体，然后再判断while循环括号内的值，只要为非0就可以进入循环体内

其中像i++、i--甚至是++i、--i这样的语句叫做自增或自减语句，如果他们单独为一个表达式语句只是对i本身做了一次加1或者减1，如果它的前面有一个等于号，结果就会不同了，谁在前面就先运行谁。如何理解这句话呢？比如int a = ++i，因为++i的加法在前，所以是i先加1再赋值给a，同样的如果是int a = i++，因为是i在前所以是先把i赋值给a，再对i做一次加1操作。

其实像这样看起来比较花哨的语法有很多个，在C++语言特性中有很多类似的东西，这些特性都是为了方便特殊做某些行业的人提供的，就像安装VS时要求你自己勾选组件一样，你可以随心所欲的选择你想用的方式将代码写好即可，只有追求极致的阅读性以及代码运行效率时才会对其做严格要求。

1.2.6 指针

这一节我们将讲解C语言的灵魂一>指针，还记得之前我们在发布我们编写好的程序时说的版本位数吗？其中就讲到了关于内存地址的东西，我们可以把内存想象成是一个很多格子组成的空间，类似于一个表格，为了方便后续理解栈空间等我们把它想象成如下这样：

内存块1地址1
内存块2地址2
内存块3地址3
内存块4地址4
内存块5地址5
内存块6地址6

每一个内存块都有相对应的地址，我们可以通过地址找到相应的内存块，这就是内存的本质，非常好理解是吧？那么这个跟指针有什么关系呢？指针就是存储的刚才上表中看到的地址1、地址2、地址3这些数据的值，我可以通过这个地址的值最后找到目标内存块1、内存块2、内存块3，这就是指针的本质，接下来请写出下面的代码：

#include"stdio.h"

int main()
{
    int x = 0;
    int* y = &x;
    *y = 3;

    printf_s("x的值为%d\n",x);

    return 0;
}

上面代码中像int就是整型指针，代表的就是指向的内存区域是代表一个整型，当然还有更为复杂的像int**，其实这些都有点算是语法糖了。本质意义并不大，我们只需要记住一个东西，当指针带有几个 *则在找寻到目标的值时就需要在前面加多少个 *，如代码中利用 *y得到目标x的空间，进而赋值为3，所以上述的代码最终运行结束后x的值为3。

相信到这里指针怎么玩你们肯定是明白了，那么一个内存空间的地址怎么取呢？请仔细看上面的代码，有块代码&x，其中在一个变量前面加上一个&就算是取到了目标变量的内存地址，变量还记得是什么吗？就是代指的某个值，类似于数学上的x、y等。所以这样就可以很方便的操纵我们的内存数据，完成我们复杂的代码编写，那么接下来我们了解了内存就不得不谈数据结构了。

1.2.7 数据结构

内存是一个庞大的空间，有一堆的格子空间供我们存放数据，如果胡乱存储找起来也是非常的麻烦，所以必须有一个数据结构能很好的把它们结构化，这样每一次找的时候就非常的方便啦。

所以聪明人们就又发明一系列的数据结构，如链表、栈、二叉树等，通过这样的结构我们在寻找目标数据时就会变得异常轻松。相信大家已经迫不及待的想学习这些结构了吧，不过在讨论他们之前我还需要介绍一个名为结构体的类型。这个家伙可厉害了，它包罗万象，任何类型的东西都可以往里面塞，塞完了就包装一下取了个别的名字。是不是听起来特别不要脸，没错！请记住这个“不要脸”的类型，它就叫做结构体，请把下面的代码编写出来：

#include"stdio.h"

struct buyaolian
{
    int x;
    char y;
};

int main()
{
    buyaolian a;
    a.x = 1;
    a.y = 'a';

    printf_s("buyaolian中a的值为%d y的值为%c\n",a.x,a.y);

    return 0;
}

是不是又发现有个叫做struct的东西，这个就是要编写结构体必不可少的东西。通过像上面这样加个struct再加个名称，再在函数体里面胡乱塞一些其它的类型就可以实现结构体的定义了，要注意一旦定义了结构体就说明你的结构体的名称就可以算做一种类型了。

我们可以看到上面通过定义结构体后，再通过其后加个 . 的方式就可以找到它的成员，进而可以对其包含的东西进行随意的操作读写。

那么到这里想必大家已经能够猜出来了，数据结构不就是这样嘛，通过编写一种特殊的结构体，然后通过其寻找并操作内存，进而实现了对整个内存空间的布局，简直太棒啦！

那么接下来就要讲几个重要的数据结构，首要的就是链表，它其实就是一个指针加数据块，还记得之前的内存布局吗？一个地址对应一个内存块，而指针只能指向目标内存地址，通过寻找就找到目标的内存数据了，这样整个过程就结束了。而链表巧妙的将指针和数据绑在一起，这样寻找到的目标数据块也包含着指针和数据，这样就可以再次往下面寻找，所以就产生了一系列的双向、单向、循环链表等，想必大家都知道是怎么回事了吧？如果还不明白自己动手画画图，看看是不是这么回事。

栈是人们规定的一种数据存储和访问的方式的结构，它比较特殊，我们还是把它想象成下面这样一个盒子

它的上面有个口，我们每次存放数据的时候就往里面放一个东西，但是取的时候肯定是先放进去后拿出来，后放进去先拿出来，想想是不是这样？但是我们知道内存肯定不是这样的对不对，内存本身它是没有口子的，所有的这一切都是人们想象的一种抽象结构。于是人们觉得这种结构非常适合现实生活存拿东西，于是人为性的把内存的一些特殊区域规定成了栈，当你压栈出栈时，就类似于现在的放东西拿东西。常见就是我们运行一些push、pop、call等汇编语句时，它直接将栈空间的栈顶进行修改。

注意：栈顶、栈底都是人为规定的一个地址，栈顶用于标记目前栈的顶部在什么位置，下一次取或者放入数据的时候栈顶会自动的调整位置

第二章 Windows逆向篇

2.1 概述

我们目前已经掌握了基本的C语法，那么目前我们可以做什么呢？就是可以很完美的调用别人写好的函数接口，可不要小看目前这个能力，即使很厉害的程序员最后也是不断的调用写好的函数接口，所以此刻你已经掌握了一个程序员基本的开发能力，但你和那些资深程序员的区别还在于经验不足。不足的地方是你对开发某些业务能使用哪些函数接口，以及出现了函数调用错误或者代码编写错误应该如何排查等问题还很欠缺。所以请不要着急，我们接下来再学习一招逆向手段。

逆向本身不是单单存在计算机行业中，它是一种思维模式。比如吃饭时尝一口就能知道这个菜放了哪些调味品，又比如听一首歌就能知道这个歌用了哪些乐器、韵律是什么样，这些通通都是通过逆向的手段反向得到最初的面貌。

2.2 逆向工具

在学习编程篇时有相应的工具，同样的在学习逆向时也有相应的工具需要准备，这些优秀的工具都是前人为我们做好方便我们逆向分析的，我们接下来就学会使用这些工具吧。

2.2.1 IDA Pro

IDA Pro这是一个非常厉害的反汇编工具，其实真正厉害并不是反汇编，而是它里面有一个插件可以实现伪反编译，我是这样理解的。而这个插件呢快捷键是F5，所以也经常被逆向圈的人叫做F5大法。一个小小的F5后，整个逆向的过程变成了读懂伪代码，逆向过程也变的如此简单了，当然这还只是较为简单的逆向，后续我会带大家慢慢接触一些较为复杂的逆向手段，现阶段就先让我们熟悉这款工具吧！

我们可以通过官网下载安装，但是所需的许可证也是一笔昂贵的费用，推荐初学时使用一些前人提供的免费版，至于免费版怎么来的？相信等你学完这个课程就可以明白啦，话不多说我们安装好后就打开这个程序吧！

IDA Pro 9.0将32位和64位融合成一个版本了，所以低版本的IDA就需要分版本打开相应的程序

我们打开这个程序后拖拽一个程序放进去，经过漫长的等待后它就会为我们分析出相应的反汇编等数据，这个工具同样很强大，不仅仅只能对exe、dll等这种PE文件进行分析，还可以对很多平台架构的各种文件进行分析，随着IDA版本的更新也会增加对各种文件的分析支持。但是由于我们目标是对Windows平台的逆向，所以我们目前就拖拽一个exe或者dll这种PE文件即可，后续我会为大家讲解PE文件是什么。

我们对比一下我们的源码

#include"stdio.h"

struct buyaolian
{
    int x;
    char y;
};

int main()
{
    buyaolian a;
    a.x = 1;
    a.y = 'a';

    printf_s("buyaolian中a的值为%d y的值为%c\n",a.x,a.y);

    return 0;
}

在这个main函数中看起来好像不大一样啊，不过不用担心，还记得我们之前说过吗？这个工具能做到的是伪反编译，并不能真正的还原成源码，只能做到近似，方便我们分析。

我们首先要清楚我们写的代码是什么，我们无非就是做了一个调用函数printf_s将我们的1和字符a打印输出了一下。那我们看看伪代码是怎么写的首先调用了一个CheckForDebuggerJustMyCode函数，接着调用了一个printf_s，整个过程与我们的代码完全相似。有人说我没有写这个CheckForDebuggerJustMyCode函数啊，这是啥时候加进去的？如果你有这种问题，请想想第一章我们讲过的计算机历史，VS是不是也是很多前人为我们做好的工具，它为了方便我们使用高级语言开发项目，所以底层的细节都替我们处理好了，它除了将你的源码做一系列的编译链接外还增加了很多安全策略等等功能，详细可以看看官方说明（/JMC（“仅我的代码”调试） | Microsoft Learn）。

所以我们就清楚了，真正送给CPU执行的代码其实就是我们反汇编看到的那些代码，我们的源码也不过只是为了方便我们开发而已。CheckForDebuggerJustMyCode函数在我们编译的时候编译器给我加上了，它有属于它自己的功能，像这种类似的函数有很多很多，我们不用关心它，我们真正需要关心的是源码写了什么。

我们发现伪代码中的printf_s输出的是一个1和97，并不是我们的1和a，这是因为ASCII码的存在，也就是我们之前说的类型，97如果不加任何说明没有人知道它是什么，但是我们在输出的时候规定的是按照%c输出，所以就按照ASCII码的规范理解成字符了。请点击97这个数字并按快捷键R看看效果，是不是字符一下子就出来了？

IDA除了可以自定义一些设置外，还可以通过添加插件插件的形式实现很多功能。插件的安装方式极其简单，大部分情况下将其放入到IDA的安装目录plugins文件夹下即可，特殊情况请参阅插件提供者说明。像IDA的背景插件，以及二进制的patch插件，再或者二进制比对插件等都是非常好用的功能，可以说就是诸多好用的插件才使得IDA功能变得愈发强大。更多好玩的插件请参照链接：ida-plugin · GitHub Topics，里面提供了大量的开源插件，寻找到适合自己的插件将其添加到自己的IDA中吧！

当然在逆向分析过程中除了静态像这样分析代码外，我们还需要通过动态调试的方式得到一些较为复杂的代码分析结果。请点击IDA中No debugger，选择当前Windows调试器Local Windows debugger，这样就将调试器切换好了。当然IDA也支持远程调试，如分析ELF文件等时，我们可以将一个ELF文件放入linux系统中，然后再放入其IDA目录下的linux_server文件，搭建好相关的配置就可以实现远程调试。

调试过程很简单，在指定的汇编窗口或者伪源代码窗口按下F2键，这样断点就已下好，当按下F9运行程序时，程序的代码运行到你下断点的位置就会停下来，这时就可以观察相应的堆栈、寄存器等信息，进而得出一些较为复杂的代码运行情况。如下图我在printf_s处下上断点，当执行到此处时程序就会停下来，再通过按下F7进入函数或F8步过函数实现调试。

相信大家都已经对IDA的基本使用有了一个较为清晰的轮廓，试着将刚才的代码重新编译成release版本，在IDA分析时不要导入PDB文件，再看看IDA的分析结果又是如何？我们会发现IDA几乎很多函数都没有标注出函数名，部分函数能够识别是因为IDA有自己内部编写好的硬编码匹配签名。大部分函数识别不了这才应该是正常的现象，想象一下程序代码都已经通过编译发布出去了，结果里面的各种信息都在，这岂不是一件很危险的事情吗？所以才需要通过逆向的手段，不断研究调试提升自己的逆向功底，进而在茫茫代码中找到真正关键的信息。

除了上述IDA这种分析工具外，像Ghidra、x64dbg等优秀的开源逆向分析工具都非常不错，各有优劣，推荐大家都在github上下载使用一下。

2.2.2 CFF Explorer

这款工具主要是对Windows平台的可执行文件进行结构化分析，上面讲IDA时有提到PE文件，这个就是Windows平台的可执行文件。我们拖拽一个案例放入其中，可以看到这款工具清晰的将文件的各种二进制信息通过PE解析形成可供查看的信息。

之前我们讲过结构体，就是将各种所谓的类型放入其中形成一种新的类型。PE结构其实就是各种结构体的组合，该一系列的结构体可以描绘出整个PE文件的各个数据都代表着什么含义，故该工具通过同样的方式进行解析，详细的PE结构请参考文档PE 格式 - Win32 apps | Microsoft Learn

这款工具通过点击左侧不同的选项可以查看相应详细的信息，并在其左侧下方提供了多个功能供其使用，如二进制编辑器、地址转换、UPX脱壳等功能。

后续将带领大家做一款简易的PE解析工具，这里只需要知道这款工具可以帮助我们分析PE结构，在以后的逆向分析中快速找到相应所需数据即可。

2.2.3 010 Editor

这款工具主要是进行二进制查看，类似的还有Winhex等工具，而这里主要推荐使用这款工具是因为其脚本库功能十分强大，可以自动进行加解密和格式化文件。我们拖拽一个exe文件放入其中，这款工具会自动识别并且利用其内置的模板存储库进行解析。

我们点击模板->模板存储库，可以看到有很多的模板库供我们选择使用，在后续学习当中有所需要的可以直接从这里下载使用。另外该程序还提供了脚本功能，有些复杂的加解密可以通过编写相应的脚本实现解密功能。

2.2.4 Detect-It-Easy

这款工具主要用来进行查壳，我们刚才已经可以看到通过IDA可以直接反编译出来我们的伪源码，这对于开发人员实在是太可怕了。所以市面上就出现了一堆这种加壳工具，利用反汇编引擎的各种特性使IDA等工具在反编译或者反汇编时出错，使得最终形成的代码根本无法正常阅读；又或者增加各种反调试功能，使其调试器在调试程序时出错，最终呈现的功能就是正常运行程序没有问题，但是对目标程序进行逆向分析时困难重重。

通过这款工具可以很清晰的查询到目标程序加的壳子到底属于哪一类，进而可以直接查询特定壳子的脱壳工具，为我们后续逆向分析提供便利。当然如果查询后发现没有任何壳子也并不代表没有壳，也有一种可能是它加了一种非常见的壳子，使得DIE的特征码未能成功匹对上。

这里我拖拽了一个加了upx壳的程序放入其中，可以看到这款工具帮我们分析出其加的是什么类型的壳子，版本号分别是多少，这样通过相应的脱壳工具就可以进行脱壳处理，可以尝试下将未脱壳的程序放入IDA中分析一下。

除了以上这些工具外还有很多好用的工具，如WireShark、Fakenet、Windbg等等，这些工具就不进行一一介绍了。大家可以直接在github网站中搜索浏览各种新鲜的工具，并阅读README文件尝试安装一下吧。
