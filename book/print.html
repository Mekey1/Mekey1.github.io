第一章 编程篇

1.1 概述

编写程序是程序员以及任何想从事计算机行业必不可免的事，它是计算机的语言，是构造现有计算机世界的一砖一瓦。而编程语言分了很多种，从层次结构可以划分为高级语言、汇编语言、机器语言等。

可是明明只有一个计算机为何有这么多语言呢？这就不得不谈历史，就像历史文字演变那样从甲骨文到金文、小篆等等，计算机也有属于它自己的历史。在最初时期我们发现可以通过操作一个庞然大物ENIAC，它是世界上第一台计算机，通过向其输入相应的数据，其内部可以产生计算后的输出结果。随后通过历史的演变，硬件以及软件的上的优化，可以发现二进制用来保存数据对硬件支持是最高效的，因为硬件可以很容易的表现出两种状态即代表了二进制的0和1，也就形成了上面所说的机器语言。

但机器语言编写十分的不便，于是“偷懒”的人们为了能够编写更快的代码就在其上又新增了一种汇编语言，按照汇编语言的格式最后编写好的代码通过一系列的工具转换后就又成了机器语言，这样确实加速了计算机的发展，而且这种“偷懒”思维也一直延续着。后续形成了像C、python、Java等各种计算机语言，这些语言就更厉害了，只需要像现在写字一样把逻辑写好，通过其编译器、链接器等工具就自动完成了一个程序的全部过程。而我们接下来将主要以C为主，因为其相对于其他语言而言“偷懒”的少，所以也可以学习到很多计算机底层逻辑。

1.2.3 代码讲解

#inlude"stdio.h"

这句代码的含义是包含了一个叫做stdio.h的头文件，我们知道程序的运行需要做很多很多的事，为了让我们能够“偷懒”，所以做出了很多的工具让我们使用包括已经写好的代码。你没错听错，就是别人写好的代码我又懒的写，所以干脆直接调用一下即可。像下面出现的printf_s函数就是别人写好的，我们直接拿来调用而已。

说的容易直接拿来用，如果你把它的代码全部拿来，又有人把你的代码拿来这样代码岂不是越来越多，这样太臃肿了，于是乎头文件的作用就是告诉程序我有这些代码，但是这些代码真正在哪里呢？这个就得有VS自己去找了，所以在VS的项目属性中包含了目标库的目录位置，以及程序运行要加载的库。

到这里我们就清楚了，要想用别人的代码到我的工程里，我可以让别人提供头文件和库，如果直接用别人的源代码实在是太臃肿了，我们要复制粘贴太多太多了。所以在我们写好#include"stdio.h"后，系统就自动去相应的目录找到目标库加载到这个工程中，进而在我调用printf_s时它就可以正常打印到窗口中。

如果让自己去实现printf_s实在是太困难了，所以有这种别人已经写好的，我们拿来直接用简直太棒了。我们只关心业务的具体实现，至于里面怎么写的我们不管，所以像这些东西通通可以叫做接口，也就是别人写好的函数，像我们听到的API等都是这样的。只不过有的接口是官方写好的，可信度以及代码的严谨性是非常高的，而有的是第三方或者个人写好的，这种安全性等都不太确信。

int main()
{
    return 0;
}

像这个形状的都叫做函数，当然如果给别人用就可以叫做接口。不过不同的是它的名字叫做main，这是个很特殊的名字，它是应用程序运行开始的函数，这都是内部已经沟通好的，所以我们在编写的时候直接用main即可。像驱动程序的入口点为DriverEntry，窗口程序的入口点为WinMain，动态链接库的入口点为dllmain等等，这些都是人们约定好的，当然如果你非要改也可以从如下图所示的位置修改入口点即可。

到这里我们就将我们写的第一个C语言程序讲解完毕了，我们应该清楚的是这个代码之所以简单是因为前人为我们做了很多事情，所以我们可以像今天这样随意敲点printf_s代码即可实现打印功能等，以后我们要做很多很多前人要做的事，这样才能对安全技术的底层有一个很深的了解。
