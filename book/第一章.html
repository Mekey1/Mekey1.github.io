第一章 编程篇

1.1 概述

编写程序是程序员以及任何想从事计算机行业必不可免的事，它是计算机的语言，是构造现有计算机世界的一砖一瓦。而编程语言分了很多种，从层次结构可以划分为高级语言、汇编语言、机器语言等。

可是明明只有一个计算机为何有这么多语言呢？这就不得不谈历史，就像历史文字演变那样从甲骨文到金文、小篆等等，计算机也有属于它自己的历史。在最初时期我们发现可以通过操作一个庞然大物ENIAC，它是世界上第一台计算机，通过向其输入相应的数据，其内部可以产生计算后的输出结果。随后通过历史的演变，硬件以及软件上的优化，可以发现二进制用来保存数据对硬件支持是最高效的，因为硬件可以很容易的表现出两种状态即代表了二进制的0和1，也就形成了上面所说的机器语言。

但机器语言编写十分的不便，于是“偷懒”的人们为了能够编写更快的代码就在其上又新增了一种汇编语言，按照汇编语言的格式最后编写好的代码通过一系列的工具转换后就又成了机器语言，这样确实加速了计算机的发展，而且这种“偷懒”思维也一直延续着。后续形成了像C、python、Java等各种计算机语言，这些语言就更厉害了，只需要像现在写字一样把逻辑写好，通过其编译器、链接器等工具就自动完成了一个程序的全部过程。而我们接下来将主要以C为主，因为其相对于其他语言而言“偷懒”的少，所以也可以学习到很多计算机底层逻辑。

这句代码的含义是包含了一个叫做stdio.h的头文件，我们知道程序的运行需要做很多很多的事，为了让我们能够“偷懒”，所以做出了很多的工具让我们使用包括已经写好的代码。你没有听错，就是别人写好的代码我又懒的写，所以干脆直接调用一下即可。像下面出现的printf_s函数就是别人写好的，我们直接拿来调用而已。

说的容易直接拿来用，如果你把它的代码全部拿来，又有人把你的代码拿来这样代码岂不是越来越多，这样太臃肿了，于是乎头文件的作用就是告诉程序我有这些代码，但是这些代码真正在哪里呢？这个就得由VS自己去找了，所以在VS的项目属性中包含了目标库的目录位置，以及程序运行要加载的库。

到这里我们就清楚了，要想用别人的代码到我的工程里，我可以让别人提供头文件和库，如果直接用别人的源代码实在是太臃肿了，我们要复制粘贴太多太多了。所以在我们写好#include"stdio.h"后，系统就自动去相应的目录找到目标库加载到这个工程中，进而在我调用printf_s时它就可以正常打印到窗口中。

如果让自己去实现printf_s实在是太困难了，所以有这种别人已经写好的，我们拿来直接用简直太棒了。我们只关心业务的具体实现，至于里面怎么写的我们不管，所以像这些东西通通可以叫做接口，也就是别人写好的函数，像我们听到的API等都是这样的。只不过有的接口是官方写好的，可信度以及代码的严谨性是非常高的，而有的是第三方或者个人写好的，这种安全性等都不太确信。

1.2.4 变量类型

上面我们写的第一个程序main函数打印hello word，其中main的前面有个int，在最后有个return 0，这都是什么东西呢？这些就是编程语言中的类型，因为其前面要求是int型也就是整型，所以后续需要通过return返回一个整型值，我这里填的是0。类型有很多种，像整型、数组类型、字符型、浮点型等。

到这里我们就都明白了，我们自己写好一个函数或者给别人用的接口，通过定义好返回的类型值这样在函数运行结束时返回给调用者就可以了。

可是，为什么有那么多类型呢？因为在计算机中同一个数据有歧义，就像我们说的汉语老子，如果我们不加任何约定就说老子，到底是指父亲呢还是指道教文化的老子呢？这个就不得而知了，所以在写好函数的时候要规定好到底返回什么类型，这样调用函数的时候才不会理解错返回值的含义。当然大部分情况是需要返回值的，如果你编写的函数不需要返回任何东西，可以使用void代替，这样在函数运行结束时不用通过return返回任何值。后续我们在编写代码时根据具体情况使用不同类型的值进行操作。

接下来请将下面的代码编写出来

#include"stdio.h"

int main()
{
    int a = 0;
    char b = 'a';
    char c[] = "123";
    float d = 1.414;

    printf_s("a=%d b=%c c=%s d=%f", a, b, c, d);

    return 0;
}

注意：变量就是我们数学中所学到的x、y，使用它代指后面赋的值，这样使用到的变量就可以直接认为是后面的值，方便记忆和操作

我们可以发现上面的代码仅仅是在之前的基础上增加了各种类型的变量， 并将其打印出来。这里小伙伴要注意，赋值在编程语言中是使用‘=’符号，将右边的数据赋值给左边的变量。所以上面我们可以看出，我们一共定义了四个变量，a、b、c、d分别是整型int、字符类型char、字符数组类型char[]、浮点类型float。具体大家可以看下表：

类型含义
int有符号整数型，平时用到的数学中的整数就可以用来存放这个
char字符类型，一次只能存放一个字符
char[]字符数组类型，什么类型的数组就可以用来存放多个同类型的数据
float浮点型，像一些小数就可以用它来存放

最后通过调用别人写好的打印函数printf_s将这些数据打印出，同样的你用人家printf_s函数，就要遵守它的规范，它明确要求了在输出内容时%d就是按照十进制输出，%c是按照字符输出，%s是按照字符串输出，%f按照浮点输出，%x按照十六进制输出等。想要学习更多官方函数的使用规范，记得参考微软的官方文档：printf_s、_printf_s_l、wprintf_s、_wprintf_s_l | Microsoft Learn

所以通过上面的两个案例我们就对整个函数以及类型有了一个清晰的认识，如果我们现在就将这个程序直接生成后发布出去可以吗？答案是不行的，因为我们的程序是debug版，其中包含了大量的调试信息，被别有用心的人拿走会很容易看出我们写的东西的是什么。在VS的上面可以通过下列框修改为release版和相应的版本位数进而发布出去。

这里肯定又要问版本位数是什么？Windows主要把版本分为32位和64位，这里我们暂且认为版本的位数对于程序能够在内存中存放数据的最大值是有关系的。之前我们已经提到利用二进制对硬件很友好，而内存上的地址需要通过数进行标记，所以32位就是2的32次方个数，这样这些地址就全部被标记好了，但是如果程序想要更大的空间32位显然不够，所以才有了后来的64位。

我们已经清楚了计算机的设计就是为了让我们能够“偷懒”，可现在这种重复性的工作又得让我们去做，实在是太傻了。所以必须有个东西能够解决重复性的问题，这个时候我们就需要再学习一种新的语法，就是循环结构。请编写好如下的代码：

#include"stdio.h"

int main()
{
	int i = 0;
	for (i; i < 10; i++)
	{
		printf_s("111\n");
	}
	
	while (i--)
	{
		printf_s("222\n");
	}

	do
	{
		printf_s("333\n");
	} while (i);

	return 0;
}

上面代码我一共写了最常用的三个循环结构，分别是for循环，while循环，do-while循环结构，还有一个奇怪的东西就是i++和i--，接下来我就一一解释这些都是什么。在for循环中，第二个分号处是判断句，如果结果非0就可以进入循环体内，也就是大括号内执行代码。while循环后面的括号只要为非0就可以进入循环体内，do-while循环先执行一次循环体，然后再判断while循环。为了方便记忆列出下表：

循环结构含义
for循环第一个分号前进行赋值，第二个分号前进行判断是否非0，非0则进入循环体，第三个分号前是循环体运行结束后要执行的内容
while循环括号内只要为非0就可以进入循环体内
do-while循环先执行一次循环体，然后再判断while循环括号内的值，只要为非0就可以进入循环体内

其中像i++、i--甚至是++i、--i这样的语句叫做自增或自减语句，如果他们单独为一个表达式语句只是对i本身做了一次加1或者减1，如果它的前面有一个等于号，结果就会不同了，谁在前面就先运行谁。如何理解这句话呢？比如int a = ++i，因为++i的加法在前，所以是i先加1再赋值给a，同样的如果是int a = i++，因为是i在前所以是先把i赋值给a，再对i做一次加1操作。

其实像这样看起来比较花哨的语法有很多个，在C++语言特性中有很多类似的东西，这些特性都是为了方便特殊做某些行业的人提供的，就像安装VS时要求你自己勾选组件一样，你可以随心所欲的选择你想用的方式将代码写好即可，只有追求极致的阅读性以及代码运行效率时才会对其做严格要求。
